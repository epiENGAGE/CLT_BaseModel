# Split model_builder_notebook.py into per-tab files

## Objective

`generic_core/examples/model_builder_notebook.py` is a single marimo notebook
that has grown to ~5,031 lines across six tabs: **Model Builder**, **Fitting**,
**Forecast**, **Export**, **Analysis**, and **Documentation**.

The goal is to split it into one file per logical section so that future edits
to a single tab only require reading and touching that one file (~200–1,800
lines) rather the entire 5,000-line file.

The entry point (`model_builder_notebook.py`) is kept as the assembled output
and is regenerated by `build_notebook.py` whenever a section file is edited.

Run the notebook the same way as before:

```
marimo run generic_core/examples/model_builder_notebook.py
```

---

## Approach: Build-time concatenation

> **NOTE:** `app.include()` does NOT exist in marimo v0.20.4 (or any released
> version as of 0.23.7). The correct approach is build-time concatenation.

Each section file (`_nb_*.py`) contains only `@app.cell`-decorated functions.
They reference the `app` variable which is defined in the assembled notebook's
header. Section files are **not standalone runnable scripts** — they are source
fragments that get concatenated by `build_notebook.py`.

`build_notebook.py` assembles them in the required include order and writes
`model_builder_notebook.py`.

### Edit workflow

1. Edit the relevant section file (e.g., `_nb_fitting.py`)
2. Run `python generic_core/examples/build_notebook.py` to reassemble
3. Run `marimo run generic_core/examples/model_builder_notebook.py`

If cells are edited via `marimo edit` in the browser, the assembled
`model_builder_notebook.py` changes. Re-split by running:
`python generic_core/examples/split_notebook.py` (Task 11 creates this tool).

## Cross-tab variable dependencies (critical)

The following variables flow **across** tab boundaries.

| Variable(s) | Defined in | Consumed by |
|---|---|---|
| helpers, imports | `_nb_shared` | everything |
| `analysis_n_metrics_input`, `analysis_metric_names`, `analysis_metric_tvs` | `_nb_analysis_metric_defs` | `_build_config` (Model Builder) |
| `config_dict`, `compartments`, `is_metapop`, `metapop_folder_input`, `metapop_travel_config`, `num_age_groups`, `num_risk_groups`, `start_date_input`, `timesteps`, `rng_seed`, `total_pop_input`, `seed_inputs`, `param_names`, `scalar_param_names`, `params_inputs`, `array_param_names`, `loaded_config`, `n_transitions`, `t_name`, `transition_vars_input`, `absolute_humidity_input`, `mobility_input`, `daily_vaccines_input`, `loaded_schedule_dfs` | `_nb_model_builder` | Shared Factory, Fitting, Forecast, Export, Analysis |
| `make_single_pop_metapop`, `make_metapop_from_folder`, `extract_history` | `_nb_shared_factory` | Fitting, Forecast, Analysis |
| `fit_result` | `_nb_fitting` | Forecast, Export |
| `forecast_result` | `_nb_forecast` | Export |

**Note on `_nb_analysis_metric_defs`:** The three cells
`_analysis_metric_defs_ui`, `_analysis_metric_sel_state`, and
`_analysis_metric_plot_controls` currently appear near the bottom of the
Analysis section in the original file (lines 4559–4609), but they must come
*before* the Model Builder cells because `_build_config` depends on
`analysis_n_metrics_input`, `analysis_metric_names`, and
`analysis_metric_tvs`.

## Required assembly order in `build_notebook.py`

```
header (docstring + app = marimo.App(...))
  → _nb_shared.py
    → _nb_analysis_metric_defs.py
      → _nb_entry.py            (tab selector, output dir, autosave)
        → _nb_model_builder.py
          → _nb_shared_factory.py
            → _nb_fitting.py
              → _nb_forecast.py
                → _nb_export.py
_nb_analysis.py    (depends on _nb_shared_factory + _nb_analysis_metric_defs)
_nb_docs.py        (depends on _nb_shared for `mo`)
footer (if __name__ == "__main__": app.run())
```

## Section file format

Each `_nb_*.py` file looks like this (NO `import marimo`, NO `app = marimo.App()`):

```python
# _nb_fitting.py
# Section: Fitting tab cells
# Part of model_builder_notebook.py — assembled by build_notebook.py
# Lines 2952–3442 of original notebook

# ---------------------------------------------------------------------------
# Fitting Tab
# ---------------------------------------------------------------------------

@app.cell
def _fitting_ui(mo, config_dict, ...):
    ...
    return (...)

@app.cell
def _run_fitting(...):
    ...
    return (fit_result,)
```
